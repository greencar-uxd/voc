## 1. 1순위 게이트 (분류 전 필수 통과)

모든 VOC는 카테고리 분류 전에 다음 질문을 먼저 통과해야 한다.

> **"UXD(화면·플로우·안내) 또는 가이드가 닿을 수 있는 문제인가?"**

- **예** → 아래 10개 카테고리로 분류 진행.
- **부분적으로 예** → 분류하되, UXD가 닿는 범위와 타 부서 의제를 분리 표기한다.
  - 카테고리 7 CS 응대/고객센터 채널 부족 → 채널 확충·응대 품질은 **CS팀** 의제
  - 카테고리 8 차량 내 매너/애완동물 동반 → 정책·제재는 **운영팀** 의제
  - 카테고리 4 주유불량 페널티 정책 개선 → 페널티/인센티브 정책 자체는 **운영팀** 의제, UXD 몫은 사전 안내 UI
- **아니오** → 카테고리 분류 대상에서 제외(범위 밖). **대시보드의 본질은 "사용자 후기 중 UX 개선·앱 안내로 사용성을 높일 수 있는 것"이다.**

### 범위 밖 (제외) — UXD가 손댈 수 없는 영역 *(2026-06-15 / 2026-06-16 사용자 확정)*
다음은 실질 불만이어도 **이 대시보드에 넣지 않는다** (운영/정책/개발 영역, 또는 무의미):
- **차량 청결·위생 결과**: 담배 냄새·찌든내, 쓰레기·꽁초, 세차 안 됨·먼지 → 운영(차량 관리). *주의: **애완동물 흔적·동반(털 등)은 카테고리 8 케이스별 IN**(6/21 #963 개털 IN · 6/22 #566 OUT) — '털=청결'이라며 자동 제외 금지. 순수 청결 결과 불만만 제외.*
- **외관 손상·정비·고장**: 스크래치·기스·찌그러짐, 브레이크/핸들 등 정비, 차량 자체 냄새
- **요금·정책**: 주행요금·쿠폰·혜택·정산 (단, 주유 페널티 사전 안내 UI 등 UXD 몫은 카테고리 4)
- **차종·공급**: 차종 다양성·완전 신규 지카존 신설 요청·**차량 대수를 더 배치/공급 요청("내 지역에 차가 없다·차 좀 늘려달라")**. ⚠ **단, 반납 시 주차 자리/공간 부족(만차)은 제외가 아니라 카테고리 9** — **주차 *공간* 부족(c9) vs 차량 *대수* 부족(OUT)** 구분 *(2026-06-30 정정 — 6/30 #448·807·510 IN vs #202·230·739 OUT. 6/29에 "배치 부족=c9"를 차 대수까지 넓힌 걸 좁힘)*.
- **버그·시스템 오류** *(2026-06-16 확정)*: 앱·서버 결함으로 동작이 깨진 것 — 사진 업로드 실패, 반납 완료 처리 안 됨, 수신/통신 오류, 화면 전환 안 됨, 앱 에러·튕김, 통신 음영(지하 LTE 안 됨)으로 인한 실패 등. **이건 UX 디자인 개선이 아니라 개발(버그 수정) 영역**이므로 제외한다. → 카테고리 2는 "절차·사진 단계가 번거로워서 **안내·디자인으로 단순화 가능한** 것"에만 한정하고, 단계 중 발생한 **오류/실패는 c2가 아니다.** **⚠ 단, 사소한 글리치/일시 오류만 제외한다. 핵심 기능이 실패해 이용 자체가 막히면(예: 앱으로 차문이 안 열려 못 탐 → 카테고리 5, 시동/반납 자체 불가 → 해당 카테고리) 포함한다** *(2026-06-17 사용자 확정 — '앱으로 차문 안 열림→원격초기화'를 사용자가 c5로 keep)*. 분류 워크플로우 결과를 '버그'라며 손으로 과제외하지 말 것.
- **물리적 접근 제약** *(2026-06-16 확정)*: 안내·화면으로 못 고치는 물리적 조건 — 예: 차가 지하 깊은 층에 있어 걸어 내려가기 멀다(차량 위치 *안내* 미비는 카테고리 1이지만, 깊이 자체는 범위 밖).
- **무의미**: 단순 칭찬("좋아요")·인사·욕설·빈값
> 교차검증 근거: 원본 설문에서 사용자 수동 선별과 대조 시, 청결·외관·요금·차종이 최대 볼륨이지만 UXD 비대상이라 전량 제외됨. 2026-06-16(270건) 사이클에선 사용자가 반납 중 시스템 오류 6건을 c2에서 전량 제외(=버그는 개발 영역). [decision-log.md](decision-log.md) 2026-06-15·06-16 사이클 참조.

## 2. 10개 카테고리 정의

| 번호 | 정식 분류명 | 정의·포함 범위 |
|---|---|---|
| 1 | 인수/대여 프로세스 불편 | 대여 시작(인수) 시점의 불편. 다층/광역 차고지에서의 차량 위치 파악, 앞 사용자 반납 위치 표기 정확도, 편도 이용 후 차량 위치 미명시 등 |

| 3 | 매너평가 플로우/사용자 평가 시스템 | 매너평가 진행 중 발생하는 문제(차문 잠김 등), 평가 기능·기준·동기 관련. ⚠ **"전 사용자가 더럽게 썼다 → 제재/신고/피드백 달라"(청결 *결과* 호소)는 c3 아님 = 제외**(청결 결과). c3는 **평가 시스템·플로우 자체의 개선 제안**에만 한정 *(2026-06-29 확정 — #798·1017·753 소거)* |

| 5 | 스마트키 개선 및 위젯 추가 | 스마트키 기능·접근성. 위젯, 상태바, 경적/비상등 UI, 어플 진입 동선 |
| 6 | 차량 사용 관련 가이드 부족 | 이용 중 차량 조작·기능에 대한 가이드 부재. **차량 기능/사양 결핍을 아쉬워하는 후기(예: 블루투스 연결 없음)도 자동 '사양 OUT' 말고 포함 검토** *(2026-06-29 #889, 케이스별)* |

## 3. 이중 카운트 룰

- 한 건의 VOC에 복수 쟁점이 섞여 있으면 **주된 마찰 1개 카테고리로 귀속**하는 것을 원칙으로 한다 (월별 카테고리 합 = 월 총계가 유지되어야 함).
- 서로 독립적인 쟁점이 분리 가능하면 건을 분리하여 각각 카운트한다.
- [확인필요] 세부 적용 기준은 원자료(원본 VOC)·사용자 확정 기준을 우선한다.

## 4. 시점 구분 룰 (반납 vs 인수)

같은 "차량 위치/반납" 소재라도 **누구의 관점인지**로 구분한다.

- **반납하는 본인의 절차 불편** → 카테고리 2 반납 사진/절차 단순화
- **앞 사용자의 반납 결과로 다음 사용자(인수자)가 겪는 불편** (차량을 못 찾음, 위치 표기 부정확 등) → 카테고리 1 인수/대여 프로세스 불편

## 5. 주유 보수성 교정

- 주유 관련 언급은 **페널티·주유 기준에 대한 불만**, 또는 **주유 미비(연료 부족 상태로 인수)에 대한 불만**일 때 카테고리 4 주유불량 페널티 정책 개선으로 분류한다. *(주유 미비 포함은 2026-06-12 사용자 확정 — [decision-log.md](decision-log.md) 참조)*
- 주유와 무관하거나 다른 쟁점이 주된 경우는 카테고리 4로 분류하지 않는다 — 과대 집계 방지를 위한 보수적 교정.
- [확인필요] 세부 교정 기준은 원자료(원본 VOC)·사용자 확정 기준을 우선한다.

## 6. 줄임말 금지 룰

- 모든 산출물(대시보드·보고서·코멘트)에서 카테고리 표기는 **"카테고리 N + 정식 분류명" 완전 표기**만 허용한다.
  - 예: "카테고리 2 반납 사진/절차 단순화" (O) / "카테고리 2", "반납 사진", "C2" (X)
- 데이터 파일 내부 키(`c1`~`c10`)는 코드 식별자로서 예외이며, 화면 표기에는 사용하지 않는다.

## 7. 수치 취급 룰

- **[배포 제약]** `data.js`에는 **집계 수치만** 포함한다. VOC 원문, 차량모델, 차고지명 등 **원본 후기 식별 가능 정보는 절대 포함하지 않는다** (공개 저장소 배포 전제). 분류 작업 중의 원문 인용은 비공개 원자료에서만 다루고, 공개되는 data.js에는 넣지 않는다.
- 확정 집계에 없는 수치를 만들지 않는다. 미상은 **"데이터 미상"** 또는 "기타(미상)"으로 표기한다.
- 계산 수치(비율·증감률)는 근거를 병기한다. 예: 65.5% = 2,108/3,218.
- 시간 축은 오직 **월 단위**. 일별·요일별·주차별 집계를 만들지 않는다.
- MoM 비교는 같은 수집 기준(regime) 안에서만 계산한다. 구간 경계 비교 금지.
- 부분 월은 "N일치 진행 중" 라벨을 붙이고, 비교는 일평균 환산으로만 한다.
