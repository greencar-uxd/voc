**단계 운영 (2026-06-17 갱신)** — 06-12~14·06-15·06-16 3연속 ≥95%로 단계 3 기준은 충족했으나, **06-17에서 일치율이 64%로 하락(원인=Claude의 수동 후보정 과적용, 분류기 자체는 91%)**. 사용자가 **"누적 정확도가 아직 높지 않다"며 단계 A 유지 선택**: 사용자가 매 사이클 직접 분류 → Claude는 대조·반영(자동 전권 보류). 재안정화(보정 후 1~2사이클 ≥95%) 시 자동 전환 재검토. 검산·사용자 최종 OK는 항상 유지. **Claude 주의: 분류 워크플로우 결과를 룰로 과하게 손대지 말 것 — 06-17처럼 과제외하면 오히려 recall이 떨어진다.**

**3단계(예외만 확인) 전환 기준 (달성):**
- 최근 **3개 사이클 연속 일치율 ≥ 95%** 이고,
- 불일치가 특정 카테고리에 쏠리지 않으면,
- → 자동 분류·반영 + **불확실 건만 플래그**하는 모드로 전환 검토.
- 단, 숫자 검산(월 합계=누적, regime 정합)과 사용자 확정 대조는 **단계와 무관하게 항상** 유지.

## 사용 방법
1. 새 데이터 분류 후, 사용자 확정치와 대조 → **불일치 건수·사유**를 위 일치율 표에 한 줄 추가.
2. 새로 내린 판단·예외는 결정 로그 표에 추가(날짜·근거).
3. 반복되는 결정은 룰셋으로, 분류 예시는 골드 예시집으로 승격.
4. 버전 날짜 갱신.
